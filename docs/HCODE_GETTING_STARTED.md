# HCode Getting Started

HCode is a security-focused IDE built on Code - OSS. This guide helps you get from first launch to a working security workflow quickly.

## First 10 Minutes

1. Open the HCode activity bar container.
2. Open Tools and run a basic command with Run Tool.
3. Open Devices and use Quick Setup Device.
4. Open ACE and verify provider and MCP status.
5. Open Bug Bounty and create a sample program.

## Core Workflows

## Tools

1. Open the Tools view.
2. Run a tool preset or custom command.
3. Use Install Tool for missing binaries.
4. Use Check All Tools to refresh availability.

## Devices

1. Run Quick Setup Device.
2. Provide host, user, auth method, and optional tags.
3. Use Test Connection after creation.
4. Use Run Command on Device for remote execution.

## ACE

1. Open ACE Dashboard.
2. Set active provider and model.
3. Run a prompt with Run ACE Prompt.
4. Use Run ACE Objective for bounded ACP runs.
5. Check MCP status and integration cards.

## Bug Bounty

1. Add Program.
2. Add in-scope and out-of-scope targets.
3. Add findings with severity and reproduction details.
4. Export report when triage is complete.

## Recommended Commands

- HCode: Open HCode Getting Started
- HCode Tools: Run Tool
- HCode Devices: Quick Setup Device
- HCode ACE: Open ACE Dashboard

## Notes

- Some provider integrations require valid credentials and endpoint configuration.
- Local CLI adapters require binaries installed on PATH.
- Remote command execution requires reachable SSH targets.
